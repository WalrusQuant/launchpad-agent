Execute a shell command in a persistent shell session.

Environment: OS=${os}, shell=${shell}, cwd=${directory}

Use this for terminal operations: git, npm, cargo, docker, running scripts, etc. Do NOT use it for file operations — use the specialized tools instead:
- Read files: use `read` (not `cat`, `head`, `tail`)
- Search contents: use `grep` (not `grep`/`rg`)
- Find files: use `glob` (not `find`, `ls`)
- Edit files: use `apply_patch` (not `sed`, `awk`, heredocs)
- Write files: use `write` (not `echo >` or heredocs)

Parameters:
- `command` (required): the shell command to run.
- `workdir` (optional): directory to run in. Use this instead of `cd dir && cmd` — never embed `cd` in the command itself.
- `timeout` (optional): milliseconds before the command is killed. Defaults to 120000 (2 minutes).
- `description` (optional): 5-10 word summary of what the command does.
- `max_output_tokens` (optional): cap on returned output. Output exceeding this or ${maxBytes} total is truncated; the full output is written to a file you can read with `read` + offset/limit.

Rules:
- Always quote paths containing spaces: `rm "path with spaces/file.txt"`.
- Run independent commands in parallel (multiple tool calls in one message) rather than chaining with `&&`.
- ${chaining}
- Use `;` only when you want to run commands sequentially regardless of failure.
- Do not use interactive flags that require input (e.g. `git rebase -i`, `git add -i`).

If you need to create a new file or directory, first verify the parent exists.
